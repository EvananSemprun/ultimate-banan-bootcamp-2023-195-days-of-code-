Relaciones entre tablas, Joins y Aliases de tablas en PostgreSQL

Relaciones entre tablas (claves primarias y foráneas)

Las relaciones entre tablas se establecen utilizando claves primarias y foráneas. Estas relaciones permiten conectar datos relacionados en diferentes tablas.

  • Clave primaria (Primary Key): Una clave primaria es un campo o combinación de campos que identifica de manera única cada registro en una tabla.
  • Clave foránea (Foreign Key): Una clave foránea es un campo o combinación de campos en una tabla que se refiere a la clave primaria de otra tabla.

Tipos de joins

PostgreSQL admite varios tipos de joins para combinar filas de diferentes tablas:

  • INNER JOIN: Combina las filas que tienen valores coincidentes en ambas tablas.
  • LEFT JOIN: Devuelve todas las filas de la tabla izquierda y las filas coincidentes de la tabla derecha.
  • RIGHT JOIN: Devuelve todas las filas de la tabla derecha y las filas coincidentes de la tabla izquierda.
  • FULL JOIN: Combina todas las filas de ambas tablas.
  • CROSS JOIN: Produce el producto cartesiano de las dos tablas.
  • NATURAL JOIN: Realiza la unión basada en columnas con el mismo nombre en ambas tablas.

Uso de aliases de tablas

Los aliases de tablas se utilizan para proporcionar nombres cortos o alias a las tablas en una consulta.

        SELECT t1.columna1, t2.columna2
        FROM tabla1 AS t1
        INNER JOIN tabla2 AS t2 ON t1.id = t2.id;

En este ejemplo, "t1" y "t2" son alias para las tablas "tabla1" y "tabla2", respectivamente. El uso de alias permite referirse a las columnas de las tablas utilizando los nombres cortos en lugar de los nombres completos de las tablas.
