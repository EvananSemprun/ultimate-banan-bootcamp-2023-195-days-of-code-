Claves Primarias y Foráneas

Una clave primaria es un atributo o conjunto de atributos que identifica de manera única a cada registro en una tabla de una base de datos. La clave primaria garantiza que no haya duplicados en la columna o conjunto de columnas designadas como clave primaria. Cada tabla en una base de datos relacional debe tener una clave primaria.

Una clave foránea es un atributo o conjunto de atributos en una tabla que establece una relación con la clave primaria de otra tabla en una base de datos relacional. La clave foránea se utiliza para mantener la integridad referencial y establecer las relaciones entre las tablas.

Normalización de Bases de Datos

La normalización es un proceso utilizado en el diseño de bases de datos para organizar y estructurar las tablas de manera eficiente, minimizando la redundancia y asegurando la integridad de los datos. El objetivo principal de la normalización es eliminar la duplicación de datos y las dependencias no deseadas.

Existen diferentes formas de normalización, como la Primera Forma Normal (1NF), Segunda Forma Normal (2NF), Tercera Forma Normal (3NF), Forma Normal de Boyce-Codd (BCNF), Cuarta Forma Normal (4NF), Quinta Forma Normal (5NF), entre otras. Cada forma de normalización tiene reglas específicas para eliminar problemas de redundancia y dependencias no deseadas en las tablas.
